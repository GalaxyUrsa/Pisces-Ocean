# 2.2 关键技术及解决途径

## 2.2.1 多源海洋再分析数据的自动化获取与管理

### 关键问题

海洋环境预报模型的训练与验证依赖高质量、长时序、多变量的海洋再分析数据。这些数据来源分散、格式各异、体量庞大（日均数百MB），人工下载效率低下，且需要精确的时间对齐与空间裁剪，难以支撑大规模模型训练。

### 解决途径

采用 **Copernicus Marine Service（CMEMS）标准化接口**，基于 `copernicusmarine` Python SDK 实现多变量数据的全自动化逐日下载与管理，统一使用 GLORYS12 全球海洋物理再分析产品（`cmems_mod_glo_phy_my_0.083deg_P1D-m`）作为数据源，涵盖以下变量：

| 数据集 | 变量 | 空间范围 | 深度范围 | 时间跨度 |
|--------|------|----------|----------|----------|
| **Glorys**（真值标签） | thetao（温度）、so（盐度）、uo（东向流速）、vo（北向流速） | 100°–160°E，0°–50°N | 0.49–651 m（33层） | 2021–2026年 |
| **SST**（海表温度） | thetao | 同上 | 仅表层（≈0.49 m） | 2021–2026年 |
| **SSS**（海表盐度） | so | 同上 | 仅表层（≈0.49 m） | 2021–2026年 |

下载工具实现了以下核心机制：

- **断点续传**：每次运行前检查本地文件是否已存在，避免重复下载
- **逐日分片**：每天独立下载为单独 NetCDF 文件，便于按日期索引与增量更新
- **统一命名规范**：文件名嵌入 `YYYYMMDD` 日期标识，支持数据加载器的自动匹配
- **异常隔离**：单日下载失败不影响整体流程，记录失败统计后继续执行

## 2.2.2 海洋三维温盐场多尺度特征提取

### 关键问题

海洋温盐场具有显著的多尺度空间结构特征：大尺度环流（数百至数千公里）决定温盐场的整体分布，中尺度涡旋（50–500公里）造成局部剧烈扰动，小尺度锋面（数公里至数十公里）则影响温跃层的精细结构。传统卷积神经网络感受野有限，难以同时捕捉跨越多个空间尺度的海洋动力特征。

### 解决途径

采用 **ConvNeXt Block** 作为基础特征提取单元，结合大核深度可分离卷积（7×7 DwConv）与现代归一化技术，显著扩大单层感受野并提升特征表达能力：

```
DwConv (7×7, groups=C)          ← 大感受野空间建模
    ↓
LayerNorm (channels_last)        ← 稳定训练，适合小批量
    ↓
Linear (C → 4C)  [PWConv1]      ← 通道维度特征交互
    ↓
GELU 激活
    ↓
Linear (4C → C)  [PWConv2]      ← 通道压缩
    ↓
Layer Scale γ (可学习, init=1e-6)  ← 稳定残差训练
    ↓
残差连接 + DropPath               ← 正则化与梯度流
```

相较于经典 ResNet Block（3×3 卷积 + BN + ReLU）：
- 7×7 深度卷积等效感受野更大，一次前向即可捕捉中尺度涡旋级别的空间相关
- LayerNorm 对批量大小不敏感，适合海洋场数据 batch_size=1 的训练场景
- GELU 激活函数较 ReLU 梯度更平滑，有助于深层训练稳定性

## 2.2.3 编解码架构与跳跃连接保留空间细节

### 关键问题

海洋环境预报本质上是一个**密集预测（Dense Prediction）**任务：输入输出具有相同的空间分辨率（480×400格点），且需要精确重建温跃层等精细垂向结构。直接使用全卷积网络难以在压缩特征的同时保留局部空间细节；而过深的全分辨率网络又导致计算代价过高。

### 解决途径

采用 **ConvNeXtUNet** 编解码架构，将 ConvNeXt Block 嵌入 U-Net 框架，兼顾全局语义理解与局部细节恢复：

```
分辨率        通道数    模块
480×400  →    64     proj_in + stage1 (ConvNeXt×2)   ──→ 跳跃连接 s1
  ↓×2
240×200  →   128     stage2 (ConvNeXt×2)              ──→ 跳跃连接 s2
  ↓×2
120×100  →   256     stage3 (ConvNeXt×3)  【瓶颈层】
  ↑×2
240×200  →   128     up1 + cat(s2) + fusion1 + stage4
  ↑×2
480×400  →    64     up2 + cat(s1) + fusion2 + stage5
               ↓
             20ch    head_temp (温度输出，20层)
             20ch    head_salt (盐度输出，20层)
```

**关键设计要点：**

1. **两级下采样（stride=2）**：编码器将空间分辨率压缩至 1/4，瓶颈层以 256 通道捕捉大尺度环流背景
2. **跳跃连接（Skip Connection）**：解码器在每个上采样步骤中拼接编码器对应层特征，融合后经 1×1 卷积压缩通道，保留温跃层空间位置等精细信息
3. **双头输出（Dual Head）**：温度与盐度分别由独立卷积头输出，避免两变量在参数上的相互干扰，同时便于分变量损失加权
4. **双线性上采样**：替代转置卷积，避免棋盘格伪影（checkerboard artifacts），海洋场连续性更好

## 2.2.4 遮蔽加权混合损失函数

### 关键问题

海洋格点数据中存在大量**陆地掩码区域**（NaN 值），若将其纳入损失计算，会引入大量无效梯度，干扰模型对海洋区域的学习。此外，温度（°C）与盐度（PSU）量纲不同、数值范围差异显著，简单求和会导致某一变量主导训练方向。

### 解决途径

设计**遮蔽加权混合损失函数（Masked Hybrid Loss）**，同时解决上述两个问题：

$$\mathcal{L} = w_T \cdot \mathcal{L}_T + w_S \cdot \mathcal{L}_S$$

$$\mathcal{L}_T = \frac{\sum_{i,j,k} \mathbf{M}_{ijk} \cdot (\hat{T}_{ijk} - T_{ijk})^2}{\sum_{i,j,k} \mathbf{M}_{ijk}}$$

$$\mathcal{L}_S = \frac{\sum_{i,j,k} \mathbf{M}_{ijk} \cdot (\hat{S}_{ijk} - S_{ijk})^2}{\sum_{i,j,k} \mathbf{M}_{ijk}}$$

其中：
- $\mathbf{M}$ 为有效海洋格点掩码（NaN 处为 0，有效处为 1），训练时动态从标签数据中派生
- $w_T = 0.9$，$w_S = 0.1$：温度权重远高于盐度，反映温度场对海洋动力过程的更强约束力
- 分母使用 `clamp(min=1)` 防止全掩码批次的除零错误

相较于普通 MSE Loss，该设计实现了：
- 海洋有效区域的无偏估计，不受陆地掩码数量影响
- 温盐物理量纲解耦，各变量按实际重要性贡献梯度

## 2.2.5 背景场时滞驱动机制

### 关键问题

实际业务海洋预报场景中，预报时刻的观测数据尚不可用，模型只能依赖**过去时刻**的数值模式背景场（Background Field）进行预报。如何在训练阶段真实模拟这一物理约束，是模型能否向业务化迁移的关键。

### 解决途径

在数据加载器中引入**固定时间偏移机制**：以目标预报日期（T 日）为基准，背景场自动读取 **T-7 天**对应的 GLORYS 再分析数据：

```python
# OceanDatasetLoader.load_single_date()
if logical_name == 'Background':
    lookup_date = datetime.strptime(date, '%Y%m%d') - timedelta(days=7)
```

这一设计的物理意义：
- **7天时效**：模拟现实业务中海洋数值预报的典型预报时效窗口
- **数据一致性**：背景场与标签场来自同一 GLORYS 产品，消除多源数据偏差，使模型专注于学习海洋动力演化规律
- **消融友好**：时间偏移量在代码中集中配置，便于开展不同预报时效（1天、3天、7天）的消融实验

## 2.2.6 Z-score 归一化与训练推理一致性保障

### 关键问题

海洋温度（约 -2°C 至 35°C）、盐度（约 30 至 40 PSU）等变量数值范围各异，且不同深度层的统计分布差异显著（深层温度接近常数，表层温度季节变化大）。直接输入原始数值会导致梯度量级失衡，训练不稳定；而归一化参数若在推理阶段丢失，则会造成系统性预报偏差。

### 解决途径

**训练阶段**：在训练开始前，遍历全部训练集样本，对每个变量（SST、SSS、bg_t_3d、bg_s_3d、label_t_3d、label_s_3d）的所有有效（非 NaN）格点计算全局均值 $\mu$ 与标准差 $\sigma$，执行 Z-score 归一化：

$$x' = \frac{x - \mu}{\sigma}$$

统计量通过两遍扫描（先求和，后求方差）避免存储全量数组，计算结果保存为 JSON 文件。

**模型保存**：归一化统计量随模型权重**一并打包进 checkpoint**（`.pth` 文件中的 `norm_stats` 键），形成"模型+统计量"的自洽单元。

**推理阶段**：从 checkpoint 恢复统计量，对输入执行同参数归一化，对输出执行反归一化，还原物理单位：

$$\hat{x} = x' \cdot \sigma + \mu$$

这一机制从根本上消除了训练-推理分布漂移（Distribution Shift）问题，保证模型在任意部署环境下的预报可靠性。

## 2.2.7 自适应学习率调度与最优模型保存策略

### 关键问题

海洋场预报任务中，损失函数在训练后期往往进入平台期（Plateau），固定学习率会导致参数在最优点附近震荡，难以进一步收敛；而保存最后一个 epoch 的模型也不能保证泛化性能最优。

### 解决途径

采用双重策略确保训练质量：

**（1）ReduceLROnPlateau 自适应学习率调度**

```
初始学习率: 1e-4
衰减因子:   factor=0.5  （触发时学习率减半）
等待轮数:   patience=10 （连续10个epoch验证损失无改善则触发）
优化器:     Adam (weight_decay=1e-5)
```

当模型陷入局部平坦区域时，学习率自动降低，使参数更新步长缩小，实现在当前最优点附近的精细调整，模拟退火式逼近全局最优。

**（2）验证集最优保存（Best Checkpoint）**

每个 epoch 结束后，对比当前验证损失与历史最优值；仅当验证损失严格下降时，覆盖保存最佳模型文件（`best_model.pth`），同步写入当前 epoch 的归一化统计量、优化器状态，支持训练中断后的断点恢复。训练全程实时绘制训练/验证损失曲线并输出为 PNG 与 CSV，便于收敛状态监控。
